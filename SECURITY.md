# Security Policy

## Supported Version

Security fixes are handled on the current `main` branch only.

## Reporting a Vulnerability

Do not open a public GitHub issue for a suspected security problem.

Please report it privately by contacting the repository owner through GitHub
or the primary private project channel used for this repository.

When reporting, include:
- a short description of the issue
- affected files or commands
- reproduction steps
- impact
- any suggested mitigation if you already have one

Please avoid posting:
- credentials or tokens
- hostnames, IPs, or private infrastructure details
- full exploit payloads in public threads

## Response Goal

Best effort targets:
- acknowledge receipt within 7 days
- provide a status update after triage
- coordinate disclosure only after a fix or mitigation is ready

## Scope Notes

This repository may contain workflow, training, and infrastructure-adjacent
code. Reports that expose secrets, unsafe defaults, path leaks, insecure
automation, or public artifact exposure are in scope.
